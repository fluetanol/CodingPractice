### 일반적인 커밋/ 양식

- [커밋 분류] : [날짜] [사이트] [문제 코드] [문제 이름]
- 만약 문제 코드가 없는 사이트라면 문제 코드를 생략해도 됩니다.
- 만약 문제 이름, 사이트 등에 띄어쓰기가 있다면 언더바를 써주세요 (단, 설명은 제외)
    
    ex : 아기_상어

### 커밋 분류

| 분류 | 설명 | 양식 | 예시 |
| --- | --- | --- | --- |
| solve | 자신이 처음으로 풀어서 정답인 코드를 올릴 경우 | [커밋 분류] : [날짜] [사이트] [문제 코드] [문제 이름]| solve : 260103 백준 2504 괄호의_값 |
| solve_other | **이미 solve 시킨 문제지만**, 다른 방법으로 추가로 풀거나, 기존 코드를 보강 및 최적화 하여 성공시킨 코드인 경우 | [커밋 분류] : [날짜] [사이트] [문제 코드] [문제 이름] | solve_other : 260103 백준 2504 괄호의_값 |
| fail | 실패한 코드지만 올리고 싶은 경우(가령 시간 복잡도 문제로 떨어진 코드라던가) | [커밋 분류] : [날짜] [사이트] [문제 코드] [문제 이름] [fail 사유] | fail : 260103 백준 2504 괄호의_값 시간초과로 떨어진 문제 |
| ref_fail | **solve시키지 못한 문제**의 다른 사람이 푼 코드를 올릴 경우 (정답 코드를 참고해서 올린 경우) | [커밋 분류] : [날짜] [사이트] [문제 코드] [문제 이름] | ref_fail : 260103 백준 2504 괄호의_값 |
| ref_solve  | **solve시킨 문제**의 다른 사람이 푼 코드를 올릴 경우 (가령 최적화 목적, 다른 방법으로 푸는 방법 모색 공부한 흔적 올리는 것) | [커밋 분류] : [날짜] [사이트] [문제 코드] [문제 이름] | ref_solve : 260103 백준 2504 괄호의_값 |
| wip | 한참 풀고 있는 중이지만 아직 작성 미 완료된 코드나 풀고 있는 과정을 커밋 하고 싶을 때 | [커밋 분류] : [사이트] [문제 코드] [설명]| wip : 백준 2504 코드작성중 (**그냥 wip를 쓰셔도 상관 없음**) |
| readme | 리드미를 수정하는 경우 | [커밋 분류] : [업데이트 한 내용 설명] | readme : 기록 추가 |
| chore | 기타 파일명 수정 등 잡일 | [커밋 분류] : [설명] | chore : 파일명 수정 |

### 코드 파일명

- **일반적인 파일명 (solve, wip)인 경우**

백준_[난이도]_[문제번호]_[문제이름]

ex: 백준_S2_1138_한줄로서기

프로그래머스_[난이도]_[문제번호]_문제이름]

ex: 프로그래머스_1_142086_가장가까운같은글자

(LV표현 생략함)

- **solve_other 인 경우**

백준_[난이도]_[문제번호]_[문제이름]_[몇 번째 풀이 인지, 1보다 큰 수여야 함]

ex : 백준_S2_1138_한줄로서기_2

- fail 인 경우

백준_[난이도]_[문제번호}_[문제이름}_fail

ex : 백준_S2_1138_한줄로서기_fail

- **ref_fail 인 경우**

백준_[난이도]_[문제번호}_[문제이름}_ref

ex : 백준_S2_1138_한줄로서기_ref

- **ref_solve 인 경우**

백준_[난이도]_[문제번호}_[문제이름}_ref

ex : 백준_S2_1138_한줄로서기_ref

### 그 외

1. 위 규칙을 목숨 걸고 지킬 필요는 없음, 그냥 나중에 커밋, 파일명 조회를 통해 풀이 내역 정보를 찾고 싶을 때 도움 되라고 지정한 거임
2. ref_solve나 ref_fail의 경우 파일 내에 타인이 푼 풀이의 참고 링크를 넣어주기를 부탁함…
3. ref_solve나 ref_fail의 경우 여러 개의 풀이가 있다면 하나의 파일 안에 잘 구분되게 넣어주길 바람…

## 업데이트

- 260218 추가 규칙

![image.png](attachment:d6cc9d9d-58db-4b77-954b-f65740ff8ec9:image.png)

이런 식으로 여러 개의 문제들을 한번에 풀고, 각각에 대한 커밋을 넣을 필요가 굳이 없다고 느껴지는 경우

```bash
solve : 260218 백준 2557, 10171, 10172, 10998 자바 기본 문제  풀이
```

이런 식으로 **문제 번호만 콤마로 표현한 후, 왜 이렇게 뭉쳐서 커밋 했는지에 대한 설명을 뒤에 쓰면 됩니다** 

- 260407 추가 규칙
1. 여러 개의 문제를 풀었는데, solve, ref_solve 등이 섞여 있다면 각각의 부류를 나누어서 커밋 할 것
2. WIP 인 문제의 경우 파일 명 앞에 WIP를 붙힐 것.

ex) WIP_백준_S2_1138_한줄로서기_2

- 260409 추가 규칙

서로 다른 언어로 같은 풀이를 풀었다면 solve_other가 아니라 solve로 취급해서 올릴 것
